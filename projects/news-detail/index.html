금융 뉴스
경제 흐름 분석 대시보드

네이버 뉴스 API로 수집한 금융 기사를 한국어 금융 특화 AI 모델(KR-FinBert)로 분류하고, 연관 종목 주가와 리스크 등급을 함께 시각화하는 Streamlit 기반 대시보드입니다.

기간 2025.03 ~ 06 규모 1인 프로젝트 스택 Python · Streamlit · HuggingFace
Background
프로젝트 배경

금융 뉴스는 매일 수백 건이 쏟아지지만, 각 기사가 시장에 긍정적인지 부정적인지 파악하려면 직접 읽어봐야 합니다. 범용 감성 분석 모델은 "손실", "하락" 같은 단어를 무조건 부정으로 분류하지만, 금융 맥락에서는 "손실 축소" "하락 폭 줄어"처럼 긍정적 신호일 수 있습니다.

이 문제를 해결하기 위해 서울대 자연어처리 연구실이 금융 데이터로 파인튜닝한 KR-FinBert 모델을 선택했고, 뉴스 수집부터 종목 주가 연동·리스크 등급 산출·보고서 내보내기까지 전체 파이프라인을 직접 구현했습니다.

src/processor.py — 리스크 등급 산출 
RISK_KEYWORDS = {
    "고위험": ["파산", "부실", "유동성 위기", "상장폐지", "영업정지"],
    "주의":   ["손실", "하락", "적자", "위험", "불확실"],
}

def classify_risk(text: str) -> str:
    for level, keywords in RISK_KEYWORDS.items():
        if any(kw in text for kw in keywords):
            return level
    return "안정"
Trouble Shooting
문제 해결 과정
PROBLEM 01
KoNLPy 설치 시 Java 의존성 오류
KoNLPy는 내부적으로 Java(JVM)를 사용하기 때문에 JDK가 없는 환경에서 ImportError가 발생했습니다. requirements.txt에 KoNLPy만 명시해두면 환경에 따라 실행이 안 되는 문제가 있어, .env.example에 JAVA_HOME 설정 안내를 추가하고 Docker 환경 가이드를 README에 포함해 해결했습니다.
PROBLEM 02
yfinance 한국 종목 티커 매핑
yfinance는 한국 주식에 ".KS"(코스피), ".KQ"(코스닥) 접미사가 필요한데, 기사에서 추출한 종목명은 한글이어서 자동 매핑이 불가능했습니다. 주요 종목 100개의 한글명→티커 딕셔너리를 직접 구축하고, 매핑되지 않는 종목은 "데이터 없음"으로 처리해 오류 없이 실행되도록 했습니다.
PROBLEM 03
Streamlit 배포 시 모델 로딩 지연
KR-FinBert 모델이 매 요청마다 로드되어 첫 실행 시 30초 이상 소요됐습니다. @st.cache_resource 데코레이터로 모델을 앱 전체에서 한 번만 로드하도록 수정해 이후 요청은 즉시 처리되도록 개선했습니다.
