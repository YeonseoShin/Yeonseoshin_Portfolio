Speakit
AI 영어 회화 튜터

영어교육 전공 수업에서 학습한 통사 구조 이론을 AI 서비스로 구현했습니다. 실시간 문법 교정과 교정 이유 설명, 복습 탭까지 갖춘 완결된 영어 학습 서비스입니다.

기간 2026.03 ~ 06 규모 1인 프로젝트 스택 Node.js · Express · OpenAI API
Background
프로젝트 배경

기존 AI 영어 앱 대부분은 틀린 문장을 수정해주지만 "왜 틀렸는지"는 설명하지 않습니다. 영어교육 전공에서 배운 통사 구조(Syntactic Structure) 이론상, 문법 오류는 범주별로 구분(불규칙 동사, 시제 오류, 관사 누락 등)해 교정하는 것이 학습 효과를 높입니다.

이 아이디어를 직접 구현하고자 OpenAI API를 연동했습니다. 단순 문장 교정이 아닌 오류 범주를 명시하고 문법 설명을 함께 제공하도록 시스템 프롬프트를 설계했습니다. API 키 보안을 위해 클라이언트 직접 호출이 아닌 Express 프록시 서버 구조를 채택했습니다.

Core Code
핵심 코드
server.js — Express 프록시 서버 
// API 키를 서버 환경변수에서만 관리
app.post('/api/chat', async (req, res) => {
  const { messages, level, topic } = req.body;

  const systemPrompt = buildSystemPrompt(level, topic);
  // 통사구조 기반 문법 교정 지시 포함

  const response = await openai.chat.completions.create({
    model: 'gpt-4o',
    messages: [
      { role: 'system', content: systemPrompt },
      ...messages
    ]
  });
  res.json(response.choices[0].message);
});
Trouble Shooting
문제 해결 과정
PROBLEM 01
Web Speech API CORS 및 브라우저 호환성
Web Speech API는 Safari에서 지원되지 않고, localhost에서 크롬도 마이크 권한이 HTTPS 없이는 제한됩니다. 개발 환경에서는 localhost 예외로 처리하고, 배포 시에는 HTTPS 필수임을 README에 명시했습니다. Safari 사용자를 위해 텍스트 입력 모드를 기본으로 유지했습니다.
PROBLEM 02
교정 파싱: AI 응답에서 교정 내용 추출
GPT 응답이 자유 형식 텍스트로 오기 때문에 교정 내용을 구조화하기 어려웠습니다. 시스템 프롬프트에 JSON 구조로 응답하도록 명시하고(corrections 배열 포함), 응답을 파싱해 교정 노트에 자동 저장하는 방식으로 해결했습니다.
